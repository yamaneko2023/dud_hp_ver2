**ボタンスタイル**:
- 3回定義されており、すべてが何らかの形で使用されている
- 単純な削除では崩れる
- 完全な依存関係の把握が必要

**レイヤー構造**:
```
基本定義（行1-3000）
  ↓ 上書き
中間調整（行3000-3900）
  ↓ 上書き
最終調整（行3900-4863）※!important多用
```

## 📝 リファクタリング手順

### Phase 1: 分析・準備（1-2週間）

#### 1-1. CSS使用状況の完全分析
```bash
# すべてのPHPファイルからclass属性を抽出
grep -r "class=" app_hp/ | grep -oP 'class="[^"]*"' > used_classes.txt

# CSSで定義されているクラスを抽出
grep -oP '^\.[a-zA-Z0-9_-]+' css/style.css > defined_classes.txt

# 未使用クラスを特定
comm -13 <(sort used_classes.txt) <(sort defined_classes.txt) > unused_classes.txt
```

#### 1-2. セレクタ重複の完全マップ
- すべての重複セレクタをリスト化
- 各定義の依存関係を図示
- 統合可能な箇所を特定

#### 1-3. CSS変数設計
```css
:root {
    /* カラー */
    --primary-100 ~ --primary-900
    --secondary-100 ~ --secondary-900
    --accent-100 ~ --accent-900

    /* スペーシング */
    --space-0 ~ --space-12

    /* タイポグラフィ */
    --font-size-xs ~ --font-size-5xl

    /* シャドウ */
    --shadow-sm ~ --shadow-2xl
}
```

### Phase 2: 段階的リファクタリング（3-4週間）

#### 2-1. セクション別リファクタリング（1つずつ）

**優先順位**:
1. ✅ リセット・ベーススタイル（影響小）
2. ✅ ユーティリティクラス（独立）
3. ⚠️ ナビゲーション（重要）
4. ⚠️ ボタン（複雑）
5. ⚠️ ヒーローセクション（重要）
6. ... 以降同様

**各セクションの手順**:
```
1. 該当セクションのスクリーンショット取得
2. 新CSSファイルに移行
3. スクリーンショット比較
4. ブラウザテスト
5. 問題なければ次へ
6. 問題あれば修正→再テスト
```

#### 2-2. !important削除戦略

**原則**: 元の定義を修正し、!importantを不要にする

```css
/* 悪い例（現状） */
.section-title-large { font-size: 4rem; }  /* 行3285 */
.vision-section .section-title-large { font-size: 5rem !important; }  /* 行3988 */

/* 良い例（修正後） */
.section-title-large { font-size: 4rem; }
.vision-section .section-title-large { font-size: 5rem; }  /* !important不要 */
```

#### 2-3. セレクタ重複統合

**ボタンの例**:
```css
/* 統合前: 3回定義 */
/* 行3152 */ .btn-primary-large { ... }
/* 行3855 */ .btn-primary-large { ... }
/* 行4055 */ .btn-primary-large { ... }

/* 統合後: 1回定義 + 必要な上書きのみ */
.btn-primary-large {
    /* 基本スタイル */
}
.vision-section .btn-primary-large {
    /* ビジョンセクション固有の上書き */
}
```

### Phase 3: 最適化（1週間）

#### 3-1. 未使用クラス削除
- Phase 1-1で特定した未使用クラスを削除
- 慎重にテスト

#### 3-2. メディアクエリ統合
```css
/* 統合前: 散在 */
@media (max-width: 768px) { .foo { ... } }  /* 行500 */
@media (max-width: 768px) { .bar { ... } }  /* 行1200 */
@media (max-width: 768px) { .baz { ... } }  /* 行2500 */

/* 統合後: 1箇所にまとめる */
@media (max-width: 768px) {
    .foo { ... }
    .bar { ... }
    .baz { ... }
}
```

#### 3-3. CSS変数置換
- すべてのハードコード色をCSS変数に置換
- 一括置換スクリプト作成

```bash
# 例: rgba(99, 102, 241, ...) → var(--primary-500)
sed -i 's/rgba(99, 102, 241/var(--primary-500/g' style.css
```

## 💡 2026年2月3日の学び

### 試みたこと
1. ✅ コメントアウト削除（成功）
2. ❌ ボタン重複削除（失敗→ロールバック）

### 学んだこと
- **段階的アプローチは正しい**が、依存関係の把握が不十分だった
- **バックアップと即座のロールバック**が重要
- **複雑な重複は予想以上に絡み合っている**
- **完全な依存関係マップ**なしでの大規模変更は危険
- **小さな成功の積み重ね**が最も安全

### 成功のカギ
1. 完全な準備（Phase 0）を省略しない
2. 自動テストとスクリーンショット比較の導入
3. 1つずつ確実に進める
4. 時間をかける（6-8週間は妥当）
